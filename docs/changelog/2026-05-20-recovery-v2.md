# 错链充值找回（Wrong-Chain Recovery）— 交付说明 v2

> 本文档合并两批改动：
> - **v1**（已上线）：拦截 + 申请流程 + 数据库
> - **v2**（最新）：状态机精简、文案/入口收敛
>
> 研发以本文档为准；v1 中所有涉及 `quoted / accepted / processing / reviewing / unrecoverable` 的逻辑均已废弃。

---

## 1. 功能目标

用户把资产打到我们**不支持的链**或**不支持的币种**时，提供一条人工找回通道：
- 链上：由运营手动桥接 / 回收
- 平台：用户提交申请 → 平台到账（扣 10% 手续费）或拒绝
- 价格策略：**固定 10% 手续费**，覆盖源链 gas + 桥费 + 人工成本，**不再走报价（quote）流程**

---

## 2. 状态机（v2 精简版）

只保留 3 个状态，全部由 **admin** 推进，用户**不能修改** `status`。

| status | 含义 | 触发方 | UI Badge |
|---|---|---|---|
| `submitted` | 用户已提交，平台处理中（含审核 + 链上找回） | 用户创建即默认 | Primary "Processing" |
| `completed` | 资金已到账用户余额 | Admin | trading-green "Completed" |
| `rejected` | 无法找回（原因写入 `admin_note`） | Admin | destructive "Rejected" |

**对比 v1：** 删除了 `reviewing / quoted / accepted / processing / unrecoverable` 5 个中间态。
**Admin 操作量：** 每笔从 3~4 次点击 → **1 次点击**（Complete 或 Reject）。

---

## 3. 数据库

### 表 `public.recovery_requests`

| 字段 | 类型 | 说明 |
|---|---|---|
| id | uuid PK | |
| user_id | uuid | 提交者 |
| tx_hash | text | 用户提供的错链交易哈希 |
| wrong_network | text | 实际打到的链 |
| wrong_token | text | 实际打到的币种 |
| claimed_amount | numeric | 用户声明金额（USD） |
| sender_address | text | 用户出金地址 |
| user_note | text | 用户备注（可空）|
| status | text | `submitted` / `completed` / `rejected`，默认 `submitted` |
| fee_percent | numeric | 默认 **10**，**不可改** |
| estimated_return | numeric | `claimed_amount × 0.9`（前端展示用，可空） |
| admin_note | text | 管理员备注（拒绝原因等） |
| processed_tx_hash | text | 平台回收/到账时的链上哈希 |
| created_at / updated_at | timestamptz | |
| quoted_at / accepted_at / completed_at | timestamptz | v1 遗留字段，**v2 仅 `completed_at` 仍写入**，其余保留兼容不再使用 |

### RLS

- **SELECT**：`auth.uid() = user_id`（用户只能看自己的）
- **INSERT**：`auth.uid() = user_id`
- **UPDATE**：admin 全权；普通用户被 trigger 拦截
- **DELETE**：无策略 = 禁止
- **ALL**：`has_role(auth.uid(), 'admin')` 全开

### Trigger `enforce_recovery_request_user_update`

- Admin 直接放行
- 普通用户 UPDATE 时：
  - 锁住所有关键字段（user_id / tx_hash / wrong_network / wrong_token / claimed_amount / sender_address / fee_percent / estimated_return / admin_note / processed_tx_hash）
  - **`status` 也禁止用户修改**（v2 新增，因为不再有 Accept/Decline 报价）

## 4. 用户端流程

### 4.1 充值前拦截 — `WalletDeposit`

底部三段独立 checkbox，**全勾才解锁地址显示**：
1. Token 是否选对
2. Network 是否选对
3. 是否使用合约地址转账

**localStorage key：** `deposit-ack-v2`（v1 老用户会重新勾一次）

底部小字入口：`Sent to wrong chain? → 申请找回`

### 4.2 申请列表页 — `/wallet/recovery`

- 顶部 intro 卡（黄色 Info 图标）
  - 标题：Wrong-chain recovery service
  - 描述：固定 10% 手续费 / 3–7 business days / **No quote needed**
- "Your requests" 列表
  - 每条：金额、token + network、status badge、时间
  - 点击进详情
- "New request" 按钮 → 表单
  - 表单字段：tx_hash、wrong_network、wrong_token、claimed_amount、sender_address、user_note
  - 表单底部预览：**You will receive ≈ $X.XX (after 10% fee)**

### 4.3 申请详情页 — `/wallet/recovery/:id`

模块顺序（v2 已删除 Quote Accept/Decline 卡）：
1. **Status Timeline**（2 步：Submitted → Completed；rejected 单独样式）
2. **Request summary**（用户填的全部字段）
3. **Fee breakdown card**（永久显示）
   ```
   Claimed amount       $X.XX
   Recovery fee (10%)  -$Y.YY
   Estimated return    $Z.ZZ
   ```
4. **Credited card**（仅 `completed` 时显示，含 `processed_tx_hash`）
5. **Admin note card**（仅 `rejected` 时显示）
6. **Support footer**：`Questions? Contact support →` 跳转项目 Discord（`discord.gg/qXssm2crf9`，新标签页）

### 4.4 其他入口

- `WalletDeposit` 底部小字
- `PendingConfirmations` 卡片底部 inline 链接（"看到错链的 tx？申请找回"）
